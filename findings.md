# 税审底稿模板分析发现

## 模板体系（2026-07-04）

### 6大业务类型
| 编号 | 业务 | 核心底稿文件 |
|------|------|------------|
| 1-1 | 企业所得税汇缴（独立纳税） | 底稿-0/1/2 三件套，163张sheet |
| 1-2 | 企业所得税汇缴（汇总纳税分公司） | 同1-1结构，分摊规则不同 |
| 2 | 研发费加计扣除 | 研发费用工作底稿52张sheet |
| 3 | 全税种核查 | 全税种核查测算底稿35张sheet |
| 4 | 财产损失审核 | 财产损失工作底稿24张sheet |
| 6 | 高新鉴证 | 高新专项审计底稿67张sheet |

### 底稿三件套分工
- **底稿-0.xlsx**: 综合类(1-01~1-17) + 资产负债审定表(SH-01~SH-17)
- **底稿-1.xls**: 损益审定表(2-01~2-16) + 收入调整(3-01) + 扣除调整(3-02)
- **底稿-2.xls**: 资产折旧(3-03) + 特殊事项(3-04~3-06) + 境外(4) + 优惠(5) + 缴纳(6) + 其他(7)

### 5类填充点
| 类型 | 示例 | 自动化策略 |
|------|------|-----------|
| A.企业信息 | 名称、行业、年度 | 直接复制 |
| B.账载金额 | 各科目账面数 | TB/科目余额表匹配 |
| C.申报表数据 | 纳税申报表各栏 | 申报表→底稿映射 |
| D.纳税调整计算 | 招待费调增=账面-允许扣除 | 规则引擎(税法公式) |
| E.审核结论 | 审核确认额 | 半自动(计算+人工) |

### 关键技术难点（原方案）
- 合并单元格密集（最高166处/表）
- 跨文件勾稽（底稿-0/1/2联动）
- 测算表固定行数（1007行）
- 版本差异（独立vs汇总纳税）

### 策略转向（2026-07-04）
放弃硬解原有模板，改为：理解税法规则 → 建立计算模型 → 生成干净底稿。
税审是规则驱动的（每条调整有固定公式），这是可编程的基础。
