# 税审底稿自动填充引擎 — 全面扩展计划

## 4大任务

### 任务A: 规则补齐（企业所得税汇缴 25条→50+条）
覆盖原底稿全部调整项：
- 收入类补齐：视同销售成本、折扣折让、专项用途财政资金支出
- 扣除类补齐：手续费佣金、党组织经费、劳保支出、差旅费、会议费等
- 资产类补齐：加速折旧明细、油气勘探、生物资产
- 特殊事项补齐：企业重组（特殊性/一般性）、政策性搬迁、新收入准则差异
- 境外税收：境外所得抵免、亏损弥补
- 税收优惠补齐：资源综合利用、所得减免、税额抵免
- 缴纳情况：所得税缴纳、汇总纳税、跨地区分摊

### 任务B: 其他业务类型（4类，各有独立规则）

#### B1-研发费加计扣除
- 规则：研发项目判定、人员人工/直接投入/折旧/无形资产摊销/设计试验/其他费用
- 核心：各费用类型占研发总费用比例上限（其他费用≤10%）
- 加计比例：100%（一般行业），120%（集成电路/工业母机）

#### B2-全税种核查
- 11个税种：增值税（销项/进项/税负）、消费税、城建税、教育费附加、印花税、房产税、土地使用税、土地增值税、资源税、环保税、车船税
- 核心：计提数 vs 缴纳数 vs 测算数 差额分析

#### B3-财产损失审核
- 损失类型：现金/应收/存货/固定资产/投资/无形资产
- 规则：清单申报 vs 专项申报、税前扣除条件

#### B4-高新鉴证
- 三维测试：研发费用占比（近三年≥3%/4%/5%）、高新收入占比（≥60%）、科技人员占比（≥10%）
- 按研发项目归集费用

### 任务C: 数据输入方式
- Excel导入：支持试算平衡表、资产台账、薪酬明细
- CSV/JSON导入
- 配置驱动的映射规则

### 任务D: 验证与对照
- 用已有底稿的填写数据作为输入
- 引擎输出 vs 手工填写 逐项对照

## 阶段拆分

### Phase A: 规则补齐
- P0: rules.py 追加25+条规则（3-01~3-06/4/5/6/7补齐）
- P1: calculator.py 适配新规则（境外抵免、亏损弥补）
- P2: workpaper_generator.py 追加sheet

### Phase B: 其他业务类型
- P0: 基础框架（BaseEngine, shared models）
- P1: 研发费加计扣除引擎
- P2: 全税种核查引擎
- P3: 财产损失审核引擎
- P4: 高新鉴证引擎

### Phase C: 数据输入
- P0: ExcelReader（TB导入）
- P1: AssetReader（资产台账）
- P2: 配置映射

### Phase D: 验证
- P0: 对照脚本
- P1: 差异报告生成
