# 审计附注自动填充引擎 (Audit Note Filler)

从决算套表 Excel 提取数据，自动填入审计报告附注 Word 模板。支持行匹配、列映射、公式稽核。

## 效果

一次跑通 45 张附注表，填了 349 个单元格。4 项主表稽核自动通过：货币资金、应收账款、固定资产、营业收入，附注数和财报数分文不差。

33 项合计字段自动计算。450 处红色占位符变黑色。

## 解决的问题

审计附注编制是每年年审最耗人的环节之一。从决算套表往 Word 模板里逐格复制粘贴，45 张表重复几百次。错一个字、串一列就要返工。

这个工具把"从哪张 sheet 的哪一列取数，填到模板的哪个位置，合计怎么算"全部写成规则，一键执行。

## 架构

```
决算套表 Excel (.xlsx)
      │
      ▼
  会计语义引擎 (SEMANTIC_TABLES)
      │  - 每张附注表的会计含义
      │  - 列映射（源列→模板列）
      │  - 行匹配（match_name 模糊匹配）
      │  - 勾稽关系（跨表校验）
      │  - 合计规则（auto_sum）
      ▼
  附注模板 Word (.docx)
      │
      ▼
  输出：已填充的附注 Word
      │
      ▼
  主表稽核（BS/IS 对比校验）
```

## 核心模块

**fill_notes.py** — 主引擎，2600 行。包含会计语义层、行匹配器、列映射器、合计计算器、主表稽核。

**fill_notes_config.py** — 配置驱动版本，适合按年复用。

**engine.py** — 底层框架：Entity 模型、模板分析器、占位符填充。

**run_pipeline.py** — 完整管线：决算套表 → 审计报告 + 底稿。

## 关键实现细节

### 行匹配 (match_name)

模板里的行名和套表里的行名很少完全一致。match_name 的匹配策略：去空格→全等匹配→括号统一→子串包含（最短 4 字）→去掉"及与和、，"再比。

v11 加固点：最短子串 5 字起，拦截不同序号前缀（如"1."和"2."不匹配），4 字公共子串不再自动接受。

### 列映射 (col_map)

模板列序和套表列序经常不一致。每张表独立配置 col_map，指定"套表列 A→模板列 3"。支持合并单元格偏移修正（T91 修复案例）。

### 合计自动计算 (auto_sum)

标记哪些行是合计项，引擎填入数据后自动求和。标记哪些行是计算项（如"账面价值 = 余额 - 坏账准备"），自动做加减法。v11 修复：小计行后的子项跳过，不参与合计计算。

### 分节感知 (section_headers)

同一个模板文件里装了多张附注表（通过分节符/分页符分隔）。引擎先扫描模板识别每个附注小节，再逐节匹配数据源。

### 主表稽核 (reconcile_with_main_statements)

填充完成后，从利润表和资产负债表提取期末数，跟附注表的关键科目金额逐一比对。不一致的输出差异表。

## 使用方式

```bash
# 单表填充
python fill_notes.py

# 在 fill_notes.py 末尾修改以下三个参数:
# 1. 决算套表路径（.xlsx）
# 2. 附注模板路径（.docx）
# 3. 输出路径
```

```bash
# 完整管线
python run_pipeline.py
# 修改 run_pipeline.py 中的 ENTITIES 配置:
# - 套表目录
# - 模板目录
# - 实体信息（企业名称、行业等）
```

```bash
# 配置驱动模式（适合按年复用）
python fill_notes_config.py
```

## 支持验证的附注表（45 项）

货币资金明细、应收票据分类、应收账款账龄/坏账准备、预付款项账龄、其他应收款账龄/坏账准备、存货分类、合同资产、固定资产/累计折旧/减值、在建工程、无形资产、短期借款、应付票据、应付账款、合同负债、应付职工薪酬（T128/T129）、应交税费、其他应付款、实收资本、资本公积、盈余公积、未分配利润、营业收入/营业成本、税金及附加、销售费用、管理费用、研发费用、财务费用、所得税费用、现金流量表（T205）、合并范围、非经常性损益、净资产收益率等。

## 已知局限

- 账龄表合计和组合表合计可能存在结构性差异（关联方组合 37M 无账龄明细）
- 依赖决算套表的固定格式 sheet 命名和列结构
- 不含直接生成审计报告全文的功能（需配合 engine.py 和 run_pipeline.py）

## 适用场景

- 会计师事务所年审附注编制
- 企业财务部门自行填报附注
- 多主体合并附注的批量处理
