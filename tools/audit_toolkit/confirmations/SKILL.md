# audit-confirmations

| 属性 | 值 |
|------|-----|
| **name** | `audit-confirmations` |

## 功能描述

审计函证管理工具，用于往来账款（应收账款/应付账款/其他往来）函证的全流程管理，包括：

1. **函证项目自动筛选** — 金额阈值 + 覆盖率双重控制，自动标记特殊项目
2. **函证控制表生成** — 16列标准化控制表，含函证编号、类型、状态追踪
3. **回函差异自动汇总** — 差异计算、分类、重大差异标红
4. **统计摘要生成** — 发函率、回函率、覆盖率、差异分布一页览

符合《中国注册会计师审计准则第1312号——函证》第十条至第十九条关于函证决策、设计、实施、评价的规范要求。

## 触发词

以下关键词触发此技能：

- **函证** — 最宽泛触发词
- **发函** — 生成函证控制表
- **回函** — 处理回函数据
- **询证函** / **往来函证** / **银行函证** — 各类函证
- **函证控制表** — 生成控制表
- **函证筛选** / **函证覆盖率** — 筛选逻辑

## 工作流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  1. 项目筛选  │───→│  2. 控制表生成 │───→│  3. 发函执行  │
│  阈值+覆盖率  │    │  编号+类型    │    │  （人工）     │
└──────────────┘    └──────────────┘    └──────┬───────┘
                                               │
┌──────────────┐    ┌──────────────┐    ┌──────▼───────┐
│  6. 审计结论  │←───│  5. 替代程序  │←───│  4. 回函追踪  │
│  形成意见    │    │  未回函项目   │    │  差异分析     │
└──────────────┘    └──────────────┘    └──────────────┘
```

### 各阶段详解

#### 阶段1: 项目筛选 (screen_items)

- 读取往来明细Excel，金额降序排列
- 特殊项目（关联方、零余额高频交易、异常标记）强制纳入函证范围
- 常规项目按金额阈值过滤（低于阈值不函证）
- 累进选取直到满足覆盖率目标（默认70%）或达到最少数量（默认10项）
- 自动分配函证类型：大额/特殊 → 积极式，小额常规 → 消极式
- 生成函证编号（格式: `AR-20260603-0001`）

#### 阶段2: 控制表生成 (export → 函证控制表)

输出16列控制表：

| 列 | 名称 | 说明 |
|----|------|------|
| A | 序号 | 流水号 |
| B | 客户/供应商名称 | 对方单位 |
| C | 账面金额 | 账面余额 |
| D | 函证金额 | 实际发函金额 |
| E | 函证类型 | 积极式/消极式 |
| F | 函证编号 | 唯一编号 |
| G | 发函日期 | 发函日期 |
| H | 回函日期 | 收到回函日期 |
| I | 回函金额 | 对方确认金额 |
| J | 差异金额 | 账面-回函 |
| K | 差异原因 | 原因分类 |
| L | 调节状态 | 待发函/已发函/已回函一致/已回函待调节/未回函 |
| M | 替代程序执行情况 | 替代测试 |
| N | 审计结论 | 最终结论 |
| O | 特殊标记 | 关联方/异常等 |
| P | 备注 | 其他说明 |

#### 阶段3: 发函执行（人工环节）

客户自行发送函证，此阶段不涉及工具操作。发函后可将发函日期填入控制表。

#### 阶段4: 回函追踪 (process_replies)

使用 `--replies` 参数传入回函数据Excel：

- 按名称匹配回函记录到控制表项目
- 自动计算差异 = 账面金额 - 回函金额
- 自动计算差异率
- 差异 > 重要性水平的项目标红
- 差异原因自动分类（尾差/少记/多记）
- 回函状态自动更新

#### 阶段5: 替代程序（人工判断+记录）

对于未回函项目，在控制表的"替代程序执行情况"列中记录：
- 检查期后收款/付款
- 检查合同、订单、发票
- 检查发货单/验收单
- 其他替代性程序

#### 阶段6: 审计结论（人工判断+记录）

综合函证结果和替代程序，形成审计结论并填入控制表。

## CLI 使用示例

### 基础用法

```bash
# 应收账款函证 — 阈值1万，覆盖70%余额
python confirmation_engine.py \
  --input "D:/审计项目/往来明细-应收账款.xlsx" \
  --output "D:/审计项目/函证控制表-应收账款.xlsx" \
  --name-col "客户名称" \
  --amount-col "期末余额" \
  --type ar \
  --threshold 10000 \
  --coverage 0.7
```

### 含回函差异分析

```bash
# 应付账款函证 + 回函数据
python confirmation_engine.py \
  --input "D:/审计项目/往来明细-应付账款.xlsx" \
  --output "D:/审计项目/函证控制表-应付账款.xlsx" \
  --name-col "供应商名称" \
  --amount-col "应付余额" \
  --type ap \
  --threshold 5000 \
  --replies "D:/审计项目/回函数据.xlsx" \
  --reply-name-col "供应商" \
  --reply-amount-col "确认金额" \
  --materiality 50000
```

### 含特殊列配置

```bash
# 其他往来款 — 含关联方、特殊标记列
python confirmation_engine.py \
  --input "D:/审计项目/其他往来.xlsx" \
  --output "D:/审计项目/函证控制表-其他往来.xlsx" \
  --name-col "对方单位" \
  --amount-col "余额" \
  --type other \
  --coverage 0.8 \
  --min-items 20 \
  --special-col "特殊标记" \
  --related-party-col "是否关联方" \
  --materiality 30000
```

### 格式化特性
- 表头蓝色加粗白字
- 冻结首行 + 自动筛选
- 金额千分位格式 `#,##0.00`
- 差异 > 重要性水平 → 红色加粗字体
- 列宽自适应（中文字符宽度感知）

## 筛选算法详解

```
输入: 所有往来项目 (all_items)
参数: 阈值T, 覆盖率C, 最少数量M

步骤:
1. 金额降序排列
2. 特殊项目（关联方/异常/零余额高频）→ 全部纳入
3. 常规项目:
   a. 金额 < T 且 已选 >= M → 停止
   b. 累进选取直到: 覆盖率 >= C 且 数量 >= M
4. 选后检查: 如数量 < M，强制追加至满足
5. 分配函证类型: 特殊/金额≥T*2 → 积极式, 其余 → 消极式
6. 生成函证编号: {AR/AP/OT}-{日期}-{序号:04d}
```

## 函证准则合规要点

本工具的设计遵循CAS 1312的以下条款：

| 准则条款 | 内容要点 | 工具实现 |
|---------|---------|---------|
| **第十条** | 函证决策 — 考虑认定层次重大错报风险 | 金额阈值 + 覆盖率 + 特殊项目标记 |
| **第十一条** | 函证对象 — 大额、账龄长、交易频繁、关联方 | 降序优先大额 + 特殊项目优先 + 关联方标记 |
| **第十二条** | 积极式/消极式选择 | 自动分配：大额/特殊→积极式 |
| **第十六条** | 函证过程控制 | 函证编号 + 发函日期 + 状态追踪 |
| **第十七条** | 未回函替代程序 | 控制表"替代程序执行情况"列 |
| **第十八条** | 回函差异调查 | 差异自动计算 + 分类 + 重大差异标红 |
| **第十九条** | 函证结果评价 | 统计摘要 + 审计结论列 |

## 常见问题

**Q: 覆盖率达不到70%怎么办？**
A: 工具会强制选取到 `--min-items` 数量。如需更高覆盖率，增加 `--min-items` 或降低 `--threshold`。

**Q: 名称匹配不上回函数据？**
A: 检查往来明细和回函数据中的名称是否完全一致（包括空格、全角/半角、括号类型等）。

**Q: 如何标记特殊项目？**
A: 三种方式：
   1. 在往来明细中添加"特殊标记"列（如"发函"），用 `--special-col` 指定
   2. 添加"是否关联方"列（值为"是"时自动标记），用 `--related-party-col` 指定
   3. 添加"异常标记"列，用 `--abnormal-col` 指定

**Q: 如何批量生成多类往来函证？**
A: 分别运行三次，传入不同的 `--type` 和 `--input`:
```bash
python confirmation_engine.py --type ar --input 应收.xlsx --output 函证-应收.xlsx ...
python confirmation_engine.py --type ap --input 应付.xlsx --output 函证-应付.xlsx ...
python confirmation_engine.py --type other --input 其他.xlsx --output 函证-其他.xlsx ...
```
