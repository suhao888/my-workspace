# 审计实质性分析程序引擎 (audit-analytics)

## 概述

本 Skill 提供一套完整的审计实质性分析程序自动化工具，满足《中国注册会计师审计准则第1313号——分析程序》的要求。引擎自动从资产负债表和利润表 Excel 文件中提取科目数据，执行四大分析模块并生成格式化的分析报告。

### 核心能力

| 模块 | 功能 | 依据 |
|------|------|------|
| **比率分析** | 20+ 财务比率自动计算 + 行业对标 | CAS 1313 第16条 |
| **趋势分析** | 多期趋势 + 环比变化率 + 异常波动标记 | CAS 1313 第14条 |
| **Benford检验** | 首位数字分布检验 + 卡方检验 + Z-score | CAS 1313 第18条 / AICPA AU-C 520 |
| **期间对比** | 本期 vs 上期 + 变动率 + 重大波动标记 | CAS 1313 第15条 |

## 适用场景

### 1. 比率分析
- **用途**：评估企业盈利能力、偿债能力、营运效率和发展能力
- **适用**：年报审计、专项审计、财务尽调
- **输出**：包含行业参考区间的比率对比表，偏离区间自动标记
- **注意**：比率计算结果需结合被审计单位实际经营情况解释；行业参考值为通用区间，不替代审计职业判断

### 2. 趋势分析
- **用途**：识别收入、成本、费用等科目的异常波动趋势
- **适用**：多期数据可用时（月度/季度/年度均可）
- **输出**：各科目逐期变化率，超过波动阈值的自动标记
- **注意**：波动阈值默认30%，对稳定行业可收紧至15%，对周期性行业可放宽至40%

### 3. Benford 定律检验
- **用途**：检测财务数据中的人为干预迹象
- **适用**：
  - ✅ 收入明细、费用明细 — 自然生成交易金额
  - ✅ 应收账款明细、应付账款明细
  - ✅ 银行存款交易记录
  - ❌ 不适用：人为设定数字（折旧率、税率、计提比例）
  - ❌ 不适用：有明确上下限的数据（身份证号、邮编）
  - ❌ 不适用：样本量 < 30 的数据集
- **输出**：首位数字分布表 + 期望值 + Z-score + 卡方统计量
- **注意**：Benford 偏离不是舞弊的充分证据，但可作为进一步审计程序的触发条件（红旗标志）

### 4. 期间对比
- **用途**：识别本期与上期的重大变动，验证管理层解释的合理性
- **适用**：有上期可比数据时
- **输出**：逐项对比 + 变动额/率 + 超重要性水平标记
- **注意**：重大波动不等于错报，需结合实质性程序验证

## 数据准备要求

### 输入文件格式
- **格式**：Excel (.xlsx)
- **BS 工作表**：行 = 科目名称，列 = 期间（默认工作表名 "BS"）
- **IS 工作表**：行 = 科目名称，列 = 期间（默认工作表名 "IS"）
- **上期文件**：格式同上（通过 `--prior-input` 指定）

### 科目名称识别
引擎内置了常见的中文科目名→标准键映射，支持以下命名变体：

```
资产总计 / 资产总额 / 总资产 → total_assets
流动资产合计 / 流动资产总计 → current_assets
营业收入 / 主营业务收入 / 营业总收入 → revenue
营业成本 / 主营业务成本 → cogs
净利润 / 税后利润 → net_profit
...
```

识别逻辑为关键词子串匹配，覆盖率达 95%+ 的常见命名方式。

## 使用示例

### 基本用法

```bash
# 执行全部分析（默认）
python analytics_engine.py -i 财务报表.xlsx -o 分析报告.xlsx

# 指定工作表名称
python analytics_engine.py -i 报表.xlsx --sheet-bs "资产负债表" --sheet-is "利润表"

# 仅比率分析
python analytics_engine.py -i 报表.xlsx -m ratios --industry 电力
```

### 含上期对比

```bash
# 包含上期数据进行期间对比
python analytics_engine.py \
  -i 2025年报表.xlsx \
  -p 2024年报表.xlsx \
  -o 2025年度分析报告.xlsx

# 自定义重要性水平（100万元）和波动阈值（20%）
python analytics_engine.py \
  -i 报表.xlsx -p 上期报表.xlsx \
  --materiality 1000000 \
  --fluctuation-threshold 20
```

### 生成图表

```bash
# 含趋势折线图和 Benford 柱状图
python analytics_engine.py -i 报表.xlsx --chart
```

### Python API 调用

```python
from analytics_engine import AnalyticsEngine

# 初始化
engine = AnalyticsEngine(
    input_path="财务报表.xlsx",
    sheet_bs="BS",
    sheet_is="IS",
    prior_input="上期报表.xlsx",   # 可选
    materiality=1_000_000,          # 重要性水平 100万
    fluctuation_threshold=30,       # 波动阈值 30%
    industry="电力",
)

# 执行全部分析
results = engine.run_all()

# 查看比率
for r in results["ratios"]["ratios"]:
    print(f"{r['指标']}: {r['显示值']} ({r['评估']})")

# 查看 Benford 检验结论
bf = results["benford"]
print(f"χ²={bf['chi_square']:.2f}, 符合Benford: {bf['is_conforming']}")

# 查看异常汇总
for a in results["summary"]["anomalies"]:
    print(f"[{a['严重程度']}] {a['来源']}: {a['描述']}")

# 生成报告
engine.generate_output("分析报告.xlsx", generate_charts=True)
```

## 行业参考值

引擎内置三种行业参考值：

| 指标 | 电力行业 | 能源行业 | 通用 |
|------|---------|---------|------|
| 毛利率 | 15-25% | 12-30% | 15-50% |
| 净利率 | 5-10% | 4-15% | 5-20% |
| ROA | 2-8% | 3-10% | 3-12% |
| ROE | 5-15% | 6-18% | 8-25% |
| 流动比率 | 1.0-2.5 | 1.0-2.5 | 1.5-3.0 |
| 速动比率 | 0.8-2.0 | 0.7-2.0 | 1.0-2.5 |
| 资产负债率 | 40-75% | 45-75% | 30-70% |
| 利息保障倍数 | 2.0-10.0 | 2.5-12.0 | 3.0-20.0 |
| 应收账款周转天数 | 60-90天 | 30-90天 | 24-90天 |
| 存货周转天数 | 18-90天 | 20-120天 | 18-90天 |
| 总资产周转率 | 0.3-0.8 | 0.3-0.8 | 0.4-1.5 |

可通过 `--industry` 参数切换：`电力`（默认）、`能源`、`通用`。

## 输出报告结构

生成的 Excel 报告包含以下工作表：

1. **比率分析** — 所有财务比率 + 计算值 + 行业参考 + 评估结论
2. **趋势分析** — 各科目逐期数据 + 环比变化率 + 异常标记
3. **Benford检验** — 首位数字分布 + 期望值 + Z-score + 卡方检验结论
4. **期间对比** — 本期 vs 上期 + 变动额/率 + 超阈值标记
5. **分析总结** — 所有异常的汇总清单（按严重程度排序）
6. **趋势图表** (可选) — 各科目趋势折线图（仅 `--chart` 时生成）

### 颜色标记规则
- 🟡 **黄色**：超过波动阈值（默认30%）— 需关注
- 🔴 **红色**：超过重要性水平（`--materiality`）— 重大异常

## 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 无异常，全部通过 |
| 1 | 存在警告级异常 |
| 2 | 存在重大异常 |

可用于 CI/CD 管道或自动化审计流程的决策。

## 注意事项

1. **分析程序 ≠ 实质性程序的全部**：分析程序提供异常信号，不能替代细节测试
2. **期望值设定**：审计师需要根据对被审计单位的了解设定合理的期望值范围
3. **调查差异**：引擎标记的异常需要在底稿中记录调查过程和结论
4. **Benford 局限**：偏离 Benford 不一定是舞弊，符合 Benford 也不一定无舞弊
5. **行业参考值**：为统计中位数区间，具体企业可能因经营模式差异而合理偏离
6. **数据质量**：GIGO 原则 — 输入数据质量直接影响分析结果有效性，建议先做数据质量检查
